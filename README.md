# Phoenix Services iOS

A native iOS rewrite of the **Phoenix Services — Flight Path Intelligence** dashboard, built completely in Swift and SwiftUI.

## 🚀 Features

- **Native MapKit Integration**: Replacing Leaflet for a high-performance, smooth, and interactive native Apple map experience.
- **SwiftUI Architecture**: Built entirely with Apple's modern declarative UI framework.
- **Glassmorphism Design**: Recreating the NetJets-inspired premium beige/cream palette using native `.ultraThinMaterial` backgrounds and effects.
- **Weather & Insight Panels**: Slide-over panels for Route Planning, Fuel Analytics, and AI Insights.
- **Dynamic Weather Overlays**: UI controls ready for hooking up to real-time precipitation, wind, and cloud MapKit overlays.

## 🛠️ Technology Stack

- **Language**: Swift 5+
- **Framework**: SwiftUI
- **Mapping**: MapKit (Apple Native)
- **Architecture**: MVVM (Model-View-ViewModel)

## 📂 Project Structure

- `ContentView.swift`: Main app layout handling the sidebar and sliding panels.
- `SidebarView.swift`: Custom navigation rail mimicking the original web app.
- `MapView.swift`: Core map rendering using `MapKit`.
- `RoutePanelView.swift`: Form inputs for departure/arrival and AI routing features.
- `WeatherModels.swift`: Data structures for flight routes and weather metrics.
- `Styles.swift`: Centralized design system (Colors and Typography).

## 🏁 Getting Started

To run the application locally on your Mac:

1. Clone this repository:
   ```bash
   git clone https://github.com/PatrickDine/Phoenix_Airspace_iOS.git
   ```
2. Open **Xcode**.
3. Create a **New iOS App Project** (SwiftUI).
4. Drag and drop all the `.swift` files from the `Sources` directory into your Xcode project navigator.
5. Delete the default `ContentView.swift` generated by Xcode.
6. Select a Simulator (e.g., iPhone 15 Pro) and press **Run (Cmd+R)**.
